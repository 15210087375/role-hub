# 行为记忆系统 - 使用指南

## 🎯 系统概述

行为记忆系统是一个**分层记忆系统**，能够从你与AI的日常交互中抽象提炼出：

- **价值观体系** - 你的核心信念和原则
- **性格特征** - 稳定的个性模式
- **心智模式** - 思考问题和处理信息的方式
- **思维模式** - 逻辑推理和决策路径
- **工作习惯** - 日常行为模式
- **行为偏好** - 选择和倾向性
- **决策框架** - 做决定的方法论
- **关注方向** - 兴趣和注意力焦点

## 🚀 快速开始

### 1. 首次使用
系统已自动安装，只需在OpenCode中使用以下命令：

```bash
/记忆系统 状态
```

### 2. 基本配置
设置你的截图路径（根据你的需求）：

```bash
/记忆系统 配置 设置 路径配置.screenshot_path "E:\\ai\\snapshot"
```

### 3. 测试系统
```bash
/记忆系统 分析 "我想设计一个分层记忆系统来记录我的思维模式"
/记忆系统 添加意图 目标与规划 "完成行为记忆系统学习" medium
/记忆系统 查看记忆 L1 3
```

## 📁 系统架构

### 核心目录结构
```
~/.behavior-memory/
├── Memory/                 # 被动沉淀轨道（自动分析）
│   ├── L0_状态层/         # 原始对话记录
│   ├── L1_情境层.yaml     # 对话场景和上下文
│   ├── L2_行为层.yaml     # 可观察的行为模式
│   ├── L3_认知层.yaml     # 思维方式和决策逻辑
│   └── L4_核心层.yaml     # 价值观和性格本质
├── Intent/                 # 主动输入轨道（你主动提供）
│   ├── 目标与规划.yaml    # 你的目标和计划
│   ├── 偏好与要求.yaml    # 明确的偏好和要求
│   └── 约束与边界.yaml    # 限制条件和边界
├── Config/                 # 设备配置层
│   ├── 设备配置.yaml      # 硬件/软件环境
│   ├── 路径配置.yaml      # 文件路径和目录
│   ├── 工具配置.yaml      # 常用工具和命令
│   └── 工作流配置.yaml    # 自动化工作流
└── Backups/                # 备份文件
```

## 📊 分层记忆说明

### L1 情境层
- **存储内容**: 对话场景、情感基调、环境信息
- **分析来源**: 从原始对话自动提取
- **使用场景**: 理解"在什么情况下说了什么"
- **查看命令**: `/记忆系统 查看记忆 L1`

### L2 行为层
- **存储内容**: 提问方式、回答偏好、用词习惯
- **分析来源**: 从L1情境和对话内容提取
- **使用场景**: 识别外在行为特征
- **查看命令**: `/记忆系统 查看记忆 L2`

### L3 认知层
- **存储内容**: 推理逻辑、决策路径、问题解决方式
- **分析来源**: 从L2行为模式抽象
- **使用场景**: 理解"如何思考"
- **查看命令**: `/记忆系统 查看记忆 L3`

### L4 核心层
- **存储内容**: 价值观、性格特质、原则信念
- **分析来源**: 从L1-L3长期聚合
- **使用场景**: 把握"你是谁"
- **查看命令**: `/记忆系统 查看记忆 L4`

## 🔄 数据流
```
你的对话 → L0原始记录 → L1情境分析 → L2行为识别 → L3认知抽象 → L4核心提炼
     ↓           ↓           ↓           ↓           ↓
  存储原始    存储情境    存储行为    存储认知    存储核心
     ↓           ↓           ↓           ↓           ↓
 可追溯性    场景理解    习惯识别    思维理解    人格把握
```

## 🛠️ 故障排除

### 常见问题

#### 1. 命令不生效
```bash
# 检查命令格式
/记忆系统 帮助

# 查看系统状态
/记忆系统 状态
```

#### 2. 配置更新失败
```bash
# 检查配置名
/记忆系统 配置 列表

# 检查键路径格式
# 正确: 配置名.键路径
# 错误: 配置名 键路径
```

#### 3. 文件权限问题
```bash
# 检查目录权限
ls -la ~/.behavior-memory/

# 如果需要，修复权限
chmod 755 ~/.behavior-memory/
```

#### 4. 编码问题（Windows）
如果看到乱码，可以：
1. 使用UTF-8编码的终端
2. 设置系统语言为中文(UTF-8)
3. 或者使用英文命令

### 数据恢复
```bash
# 从备份恢复
/记忆系统 恢复 backup_20250308.json

# 如果系统损坏，可以删除配置文件重新初始化
rm -rf ~/.behavior-memory/Config/*.yaml
# 下次启动时会创建默认配置
```

## 🔮 高级用法

### 自定义分析规则
你可以编辑 `~/.behavior-memory/persona_core.py` 中的分析函数来自定义：

1. `_detect_context_type()` - 情境类型检测
2. `_detect_emotional_tone()` - 情感基调检测
3. `_detect_question_pattern()` - 提问模式检测
4. `_extract_word_preferences()` - 用词偏好提取

### 集成到工作流
```bash
# 在脚本中使用
python ~/.behavior-memory/persona_core.py --status
python ~/.behavior-memory/persona_core.py --config 路径配置.screenshot_path="E:\\ai\\snapshot"
python ~/.behavior-memory/persona_core.py --export backup.json
```

### 自动化分析
你可以创建脚本定期分析对话历史：
```python
from persona_core import BehaviorMemoryCore

core = BehaviorMemoryCore()
# 批量分析历史对话
# 生成人格报告
# 发送分析结果到邮箱等
```
