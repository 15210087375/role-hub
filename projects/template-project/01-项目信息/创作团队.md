# 创作团队协作规范

## 角色职责明细

### 架构师职责
1. **项目启动阶段**
   - 填写项目信息（project-info.md）
   - 建立世界观基础设定
   - 创建主要角色库（10-15个核心角色）
   - 规划第一卷大事件时间线
   - 输出主线大纲和分卷规划

2. **写作支持阶段**
   - 维护角色库和地点时间线
   - 提供每章写作前的角色状态摘要
   - 整合主笔反馈的实际变化
   - 规划后续卷的大事件

3. **质量控制阶段**
   - 检查角色一致性（主要角色）
   - 审核时间线合理性
   - 确保世界观逻辑自洽
   - 提供剧情发展建议

### 主笔职责
1. **写作准备阶段**
   - 阅读架构师提供的角色状态摘要
   - 查询角色库获取必要信息
   - 明确本章目标和爽点设计

2. **写作执行阶段**
   - 按口语化技巧创作正文
   - 记录写作中的实际变化
   - 确保角色行为符合性格设定
   - 控制章节长度和节奏

3. **写作完成阶段**
   - 提交章节正文文件
   - 提交变化记录文件
   - 进行自检（苏格拉底提问法）
   - 等待责编检查反馈

### 责编职责
1. **质量检查阶段**
   - P0检查：世界真实性、角色一致性、毒点雷点
   - P1检查：口语化质量、节奏爽点、读者适配
   - P2检查：语言润色、细节优化、文学性提升

2. **问题反馈阶段**
   - 标注具体问题章节和段落
   - 提供可行的修改建议
   - 不代写核心剧情，只指导修改

3. **验收记录阶段**
   - 记录检查结果和修改建议
   - 标记章节状态（通过/需修改/拒绝）
   - 反馈给主笔和架构师

### 设定师职责
1. **资产设计阶段**
   - 接收架构师的功能需求
   - 设计具体角色/物品/系统资产
   - 确保资产不OOC，有使用限制

2. **出库审核阶段**
   - 审核主笔的出库申请
   - 裁定：approved / rejected / fallback
   - 提供审核理由和替代建议

3. **库维护阶段**
   - 维护资产库的完整性和一致性
   - 更新资产状态和版本
   - 提供资产使用指南

## 协作流程

### 标准写作流程
```
架构师规划 → 提供基础库和摘要
      ↓
主笔写作 → 查询库信息 + 记录变化
      ↓
责编检查 → P0/P1/P2分级检查
      ↓
架构师更新 → 整合变化，更新正式库
```

### 问题处理流程
```
发现问题 → 记录到08-协作记录/问题追踪.md
      ↓
分类处理 → P0立即改，P1建议改，P2酌情改
      ↓
分配责任 → 明确负责人和解决时限
      ↓
跟踪解决 → 定期检查问题状态
```

### 版本控制流程
```
日常开发 → 在feature/项目名分支
      ↓
章节完成 → 提交到dev分支
      ↓
责编通过 → 标记章节状态
      ↓
卷完成 → 合并到main分支
```

## 沟通规范

### 会议制度
- **周会**: 每周一次，同步进度和问题（30分钟）
- **紧急会议**: 发现P0级问题时立即召开
- **评审会**: 每卷完成后进行整体评审

### 文档规范
- **文件命名**: 统一使用中文，明确含义
- **版本标记**: 重要文件标注版本和日期
- **更新记录**: 每次修改记录修改内容和原因

### 沟通工具
- **主要沟通**: Git提交和Issue跟踪
- **即时沟通**: [指定即时通讯工具]
- **文件共享**: Git仓库，不通过其他渠道

## 质量标准

### 必须达到（P0）
1. **无逻辑矛盾**: 角色身份、行为、地点无前后矛盾
2. **无毒点雷点**: 不触碰网文常见雷区
3. **世界真实**: 避免明显的主角特权痕迹
4. **角色一致**: 行为符合性格，变化有铺垫

### 应该达到（P1）
1. **口语化达标**: AI味不明显，更像人类书写
2. **节奏合理**: 每章有爽点，压抑释放平衡
3. **读者适配**: 语言复杂度匹配目标读者
4. **信息清晰**: 关键信息明确，不模糊

### 争取达到（P2）
1. **文学性**: 有一定文笔和感染力
2. **创新性**: 有独特创意和亮点
3. **深度**: 有一定思想深度和情感共鸣
4. **完整性**: 结构完整，伏笔回收

## 绩效评估

### 量化指标
- **架构师**: 角色库完整度、时间线准确性、规划合理性
- **主笔**: 日更字数、口语化达标率、变化记录及时性
- **责编**: 问题发现率、修改建议采纳率、检查及时性
- **设定师**: 资产设计质量、出库审核准确性、库维护完整性

### 质量指标
- **读者反馈**: 留存率、订阅率、评论质量
- **平台数据**: 点击量、收藏量、推荐票
- **团队评价**: 协作顺畅度、问题解决效率

## 冲突解决

### 常见冲突类型
1. **创作理念冲突**: 对剧情发展方向有分歧
2. **质量标准冲突**: 对什么是"合格"有不同理解
3. **责任边界冲突**: 对谁该负责什么有争议
4. **进度压力冲突**: 时间紧张导致质量与进度矛盾

### 解决原则
1. **读者优先**: 以读者体验为最高标准
2. **数据说话**: 用读者数据和市场反馈做决策
3. **专业尊重**: 各角色专业领域内尊重专业判断
4. **妥协艺术**: 在关键问题上坚持，在次要问题上妥协

### 解决流程
```
提出冲突 → 记录到问题追踪
      ↓
分析原因 → 明确冲突本质
      ↓
协商方案 → 基于原则提出解决方案
      ↓
决策执行 → 负责人做出最终决策
      ↓
跟踪效果 → 观察解决方案效果
```
