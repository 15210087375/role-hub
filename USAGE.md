# 网文创作协作系统使用指南

## 系统概述

这是一个完整的网文创作协作系统，包含三个核心角色（架构师、主笔、责编）和标准化的项目模板结构。系统旨在提高创作质量，确保角色一致性、世界真实性和去AI味。

## 核心角色

### 1. 架构师 (Novel Architect)
**职责**：
- 世界观设定和维护
- 主要角色设计
- 时间线规划
- 力量体系设计
- 防"世界静止"监控

**技能触发**：`小说架构师模式`、`novel architect`、`剧情架构`

### 2. 主笔 (Creative Writer)
**职责**：
- 章节正文创作
- 次要角色记录
- 口语化执行
- 人类思维模拟
- 变化记录

**技能触发**：`小说主笔`、`写正文`、`章节创作`、`creative writer`

### 3. 责编 (Story Editor)
**职责**：
- 质量检查
- 角色一致性验证
- 世界真实性检查
- 去AI味审核
- 协作接口管理

**技能触发**：`小说责编`、`修改润色`、`章节审阅`、`story editor`

## 快速开始

### 1. 创建新项目

```bash
# 进入工具目录
cd D:\devTools\ai\opencode\role-hub\tools

# 创建新项目
python project-init.py 我的小说项目

# 或指定目标目录
python project-init.py 我的小说项目 --target D:\我的作品
```

### 2. 填写项目信息

编辑 `projects/我的小说项目/01-项目信息/project-info.md`，填写：
- 作品名称、类型、目标读者
- 核心梗概、主题思想
- 预期篇幅、更新频率

### 3. 设计世界观

架构师进入`小说架构师模式`，在`02-世界观/`目录中：
1. 创建世界观设定
2. 设计势力系统
3. 制定力量体系

### 4. 创建角色库

在`03-角色库/`目录中：
1. 使用`角色模板.md`创建主要角色档案
2. 更新`index.yaml`角色索引
3. 绘制`角色关系图.md`

### 5. 规划时间线

在`04-时间线/`目录中：
1. 创建`大事件时间线.yaml`
2. 使用时间计算工具：
```bash
cd projects/我的小说项目/04-时间线
python 时间计算工具.py "元年3月" "二年6月" --events
```

### 6. 制定大纲

在`05-大纲/`目录中：
1. 创建`主线大纲.md`
2. 制定分卷规划
3. 编写章节细纲

### 7. 开始写作

主笔进入`小说主笔模式`，在`06-正文/`目录中：
1. 创建新章节文件
2. 遵循写作指导模板
3. 记录变化到`08-协作记录/变化记录模板.md`

### 8. 质量检查

责编进入`小说责编模式`，执行检查：
```bash
cd projects/我的小说项目
python ../tools/consistency-check.py
```

## 工具使用

### 项目初始化工具
```bash
python tools/project-init.py [项目名称] [选项]
选项：
  --template, -t  模板目录路径
  --target, -d    目标目录路径
```

### 角色查询工具
```bash
python tools/character-query.py [角色名称] [选项]
选项：
  --project, -p   项目目录路径
  --no-relations  不显示关系
  --no-timeline   不显示时间线
  --no-changes    不显示变化记录
  --verbose, -v   详细输出
  --json, -j      输出JSON格式
```

### 一致性检查工具
```bash
python tools/consistency-check.py [选项]
选项：
  --project, -p   项目目录路径
  --show-info, -i 显示提示信息
  --json, -j      输出JSON格式
  --category, -c  只检查特定类别
                  (character/world/timeline/anti-ai/collaboration/all)
```

### 时间计算工具
```bash
cd projects/项目名称/04-时间线
python 时间计算工具.py [开始时间] [结束时间] [选项]
选项：
  --events, -e    显示期间的事件
  --format, -f    输出格式 (days/months/years)
```

## 协作流程

### 标准协作流程
```
架构师规划 → 提供基础库和摘要
     ↓
主笔写作 → 查询库信息 + 记录变化
     ↓
责编检查 → P0/P1/P2分级检查
     ↓
架构师更新 → 整合变化，更新正式库
```

### 变化记录规范
主笔在写作过程中必须记录：
1. **新增内容**：新角色、新地点、新设定
2. **修改内容**：角色性格变化、地点发展、设定调整
3. **删除内容**：废弃的设定、不再使用的角色

### 问题追踪规范
责编发现的问题按优先级分类：
- **P0 (必须修复)**：逻辑矛盾、角色OOC、世界真实性破坏
- **P1 (建议修复)**：一致性小问题、AI味较重
- **P2 (可选修复)**：文字润色、细节优化

## 质量控制标准

### 去AI味标准
1. **口语化表达**：避免书面语，使用自然对话
2. **短段落**：每段不超过8行，提高可读性
3. **少修饰词**：减少"非常、极其、特别"等修饰词
4. **不用破折号**：避免使用破折号进行解释说明
5. **人类思维**：增加犹豫、矛盾、不完美的思考过程

### 角色一致性标准
1. **性格动机**：行为必须符合性格和动机
2. **成长变化**：角色应随时间合理成长
3. **关系网络**：关系应有来有往，避免单向
4. **避免完美**：角色应有缺陷和弱点
5. **避免纯恶**：反派应有合理动机

### 世界真实性标准
1. **动态世界**：地点、势力应随时间变化
2. **合理奇遇**：奇遇不能是主角专属，其他人也可获得
3. **经济系统**：货币、资源应有合理流通
4. **社会结构**：阶级、权力应有合理分布
5. **时间流逝**：时间应有真实感，避免"世界静止"

## 常见问题

### Q: 如何切换角色模式？
A: 使用对应的技能触发词：
- 架构师：`小说架构师模式`
- 主笔：`小说主笔`
- 责编：`小说责编`

### Q: 项目初始化失败怎么办？
A: 检查：
1. 模板目录是否存在：`projects/template-project`
2. 目标目录是否有写入权限
3. 项目名称是否已存在

### Q: 一致性检查报错如何处理？
A: 根据错误级别处理：
- **错误 (error)**：必须修复，否则无法继续
- **警告 (warning)**：建议修复，影响质量
- **提示 (info)**：可选择性优化

### Q: 如何添加新工具？
A: 在`tools/`目录中创建新的Python脚本，遵循现有工具的结构。

## 最佳实践

### 写作最佳实践
1. **每日记录**：每天结束前记录当天的变化
2. **定期检查**：每写完3章执行一次一致性检查
3. **角色查询**：写作前查询相关角色信息
4. **时间计算**：涉及时间跨度时使用时间计算工具
5. **协作沟通**：发现问题及时记录到问题追踪

### 协作最佳实践
1. **明确职责**：各角色专注自己的职责范围
2. **及时反馈**：发现问题立即记录，不拖延
3. **版本控制**：每次修改提交有意义的commit message
4. **定期同步**：每天同步到远程仓库
5. **备份重要**：定期备份重要文件
